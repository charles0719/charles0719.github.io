Claude Code 插件体系：Marketplace、Plugin、Skill 三层关系梳理

发表于 2026-04-04 分类于 技术分享 本文字数： 1.9k 阅读时长 ≈ 2 分钟

最近在整理自己的 Claude Code 插件仓库，顺便把 Marketplace、Plugin、Skill 这三层关系梳理一下，官方文档写得比较分散，这里做个简单总结。

一句话概括

Marketplace 是仓库，Plugin 是包，Skill 是功能。

一个 Marketplace（Git 仓库）里可以包含多个 Plugin，每个 Plugin 下面又可以有多个 Skill。Claude Code 通过这三层结构来组织和分发扩展能力。

三层结构关系

每一层是什么

Marketplace —— 分发渠道

Marketplace 就是一个 Git 仓库，充当”应用商店”的角色。一个 Marketplace 下可以放多个 Plugin。

添加 Marketplace：

添加之后 Claude Code 会把仓库克隆到本地缓存，你就可以从里面安装 Plugin 了。

Plugin —— 能力包

Plugin 是一组相关 Skill 的集合，通过 plugin.json 声明元信息（名称、描述、作者）。

安装 Plugin：

安装后，这个 Plugin 下的所有 Skill 就可以被 Claude Code 使用了。

Skill —— 具体能力

Skill 是最小的功能单元，每个 Skill 通过 SKILL.md 定义：

  • 触发条件：什么情况下自动激活（关键词匹配、用户意图识别等）
  • 行为描述：激活后 Claude 应该怎么做
  • 参考资料：references/ 目录下的文档，作为 Skill 执行时的知识库

大部分 Skill 是自动触发的，不需要手动调用。比如当你问”如何使用 Claude Agent SDK”，对应的 Skill 会自动激活。少数 Skill 通过斜杠命令手动调用，如 /session-harvest。

实际例子

以 any-agent 这个 Marketplace 为例：

Plugin 包含的 Skill 说明
agent-sdk-guide claude-agent-sdk、agent-sdk-custom-tools、agent-sdk-sessions、agent-sdk-advanced Agent SDK 使用指南，按主题拆分为 4 个 Skill
session-harvest session-harvest 从会话中提取知识，生成可复用的 Skill
claude-plugin-marketplace-setup claude-plugin-marketplace-setup 教你如何创建自己的 Marketplace

可以看到，一个 Plugin 可以只有一个 Skill（如 session-harvest），也可以有多个 Skill（如 agent-sdk-guide 下有 4 个）。拆分粒度取决于功能的复杂度和关联性。

类比理解

如果用手机应用商店来类比：

概念 类比 说明
Marketplace App Store / 应用商店 分发渠道，可以有多个
Plugin App / 应用 安装/卸载的基本单位
Skill App 内的功能模块 安装 App 后自动获得的具体能力

如何创建自己的 Marketplace

如果你也想整理自己的插件，目录结构按上面的规范来就行。核心文件：

  • 根目录 .claude-plugin/marketplace.json：声明这是一个 Marketplace
  • 每个 Plugin .claude-plugin/plugin.json：声明插件元信息
  • 每个 Skill SKILL.md：定义触发条件和行为

具体的 schema 可以参考 claude-plugin-marketplace-setup 这个 Skill，里面有完整的规范说明。
